# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A Linux-only desktop wrapper for WhatsApp Web, built with **Tauri 2 (Rust-only, no JS frontend build)**.
There is no web frontend to compile: `frontendDist` (`src-tauri/dist/index.html`) is a stub. The real UI is
**remote** — `https://web.whatsapp.com` is loaded as an external URL into the main webview, and the Rust side
augments it via injected scripts and an IPC bridge.

## Commands

All `cargo`/`tauri` commands run from **`src-tauri/`**, not the repo root.

```bash
cd src-tauri
cargo tauri dev            # run with hot-rebuild
cargo build                # debug binary -> target/debug/whatstauri
cargo tauri build          # bundles -> target/release/bundle/{rpm,deb,appimage}
cargo tauri build --bundles rpm   # single bundle type

cargo test                            # all tests (unit tests are inline #[cfg(test)] modules)
cargo test nvidia_vendor_detection    # run a single test by name

# CI gates (run these before pushing — they block merge):
cargo fmt --check
cargo clippy --all-targets -- -D warnings
```

Build-time system deps (Fedora) are in `README.md`; CI uses the apt equivalents in `.github/workflows/ci.yml`.

## CI and release flow

- **`ci.yml`** (on push to `main`/`feature/**` and all PRs): `cargo fmt --check`, `cargo clippy -D warnings`,
  `cargo test`, `cargo build --release`, plus a **version-consistency** job asserting the version in
  `src-tauri/tauri.conf.json` **equals** the one in `src-tauri/Cargo.toml`. Keep them in sync or CI fails.
- **`tag-on-version.yml`** (on push to `main`): reads `version` from `src-tauri/tauri.conf.json`; if tag
  `v<version>` does not already exist, it waits for CI, creates the tag, and triggers `release.yml` (builds
  rpm+deb, publishes a GitHub Release). **Merging to `main` without bumping the version cuts no release.**
  To ship: bump the version in **both** `tauri.conf.json` and `Cargo.toml`, then merge.

## Architecture

**Startup order matters** (`src/lib.rs` `run()`):
1. `Config::load()` then `cfg.apply_env_workarounds()` — sets `WEBKIT_*` / NVIDIA env vars and **must run
   before any GTK/WebKit initialization** (it sets process env that WebKit reads at init).
2. Tauri `Builder` with plugins: `single-instance` (focuses existing window on relaunch), `opener`, `dialog`.
3. `setup`: `wipe::handle_pending_wipe()` runs **before** the webview opens its storage → tray → main window
   → first-run GNOME tray hint.

**Module map** (`src/`):
- `config.rs` — TOML config (`~/.config/whatstauri/config.toml`, all keys optional, never refuses to start),
  the `WEBKIT_*` env escalation flags, and NVIDIA detection (`/sys/class/drm` vendor `0x10de`).
- `window.rs` — the core. Builds the webview pointed at WhatsApp Web; routes links; handles downloads;
  installs crash recovery.
- `notifications.rs` — IPC command handlers + native notifications + unread state.
- `unread.rs` — parses the unread count out of the document title.
- `links.rs` — URL classification (`is_internal`, `is_browser_wall`).
- `tray.rs` — tray icon/menu and unread badge; GNOME AppIndicator hint.
- `wipe.rs` — "Clear data & re-login": deletes webview storage, preserves the config file.

**The Rust ↔ web bridge.** Two scripts in `inject/` are injected as webview initialization scripts
(`window.rs`): `notification-shim.js` replaces `window.Notification` and forwards to Rust; `title-watcher.js`
reports the title (Rust parses the unread count) and holds a `navigator.locks` lock so WebKitGTK does not
throttle the hidden page. Only the **top frame** talks to Rust; iframe copies exit immediately.

**IPC commands must be declared in three places that stay in sync** — adding/renaming one means editing all:
1. `build.rs` — `AppManifest::new().commands([...])`
2. `src/lib.rs` — `invoke_handler!` 
3. `capabilities/remote.json` — `allow-*` permissions

`remote.json` is what grants a **remote origin** (`https://web.whatsapp.com`, scoped to that URL only)
permission to invoke these three commands — nothing else can. After changing commands, rebuild and **commit
the regenerated `src-tauri/gen/` and `src-tauri/permissions/autogenerated/`** files.

## Linux/WebKitGTK specifics

- **Webview storage** lives under the app's `app_local_data_dir` (`~/.local/share/com.shatteredsun.whatstauri/`).
  This path must never change once released — user sessions live there. The config file is deliberately separate
  and survives a wipe.
- **Crash recovery** hooks the raw `webkit2gtk` `web_process_terminated` signal (Tauri does not expose it),
  reloading up to 3×/min before giving up. The `webkit2gtk` dep and this code are target-gated to Linux.
- **NVIDIA auto-quirks** (`nvidia_quirks`, default on): when an NVIDIA GPU is detected, sets
  `__NV_DISABLE_EXPLICIT_SYNC=1` on Wayland (WebKit bug 280210) and demotes NVDEC via
  `GST_PLUGIN_FEATURE_RANK` so video decodes via openh264. Existing env vars are respected, never overridden.
  NVIDIA + X11 cannot render accelerated at all — Wayland is required.
- **Link/window policy**: in-app navigation stays in-app only for internal URLs; everything else (and every
  `target="_blank"`/`window.open`, which is always denied) opens in the system browser. Downloads go to
  `~/Downloads`; the code trusts the filesystem over wry's unreliable `success` flag.

## Icons

Bundled icons in `src-tauri/icons/` must be **8-bit RGBA PNG** — Tauri's bundler rejects non-RGBA (e.g. palette)
PNGs at build time. The same bundled icon drives the window icon and the **tray** icon (`tray.rs` uses
`default_window_icon()`). On **Wayland the taskbar icon is not set by the app** — it is resolved from
`WhatsTauri.desktop`'s `Icon=whatstauri` via the installed icon theme, so taskbar icon changes ship through the
rpm/deb bundle (which installs into the hicolor theme), not through app code.
